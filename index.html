01 — Philosophy

Inspired by OpenClaw.
Built from scratch.

OpenGhost follows the core idea that a capable AI agent doesn't need a complex orchestration framework — it needs a well-designed workspace and clear instructions. The foundation is a single markdown file. Everything else follows from that.

01

Context by Reference

The agent reads what it needs, when it needs it. No token bloat. No injecting the entire world into every prompt.

02

Files Over RAM

Memory that matters goes to disk. Mental notes don't survive restarts. Files do. Text beats ephemeral memory every time.

03

Operator-Defined Identity

The agent's personality, values, and rules are written by the operator in plain markdown — not hardcoded into the engine.

04

Real Access, Clear Guardrails

Full device access with safety from explicit rules in AGENTS.md — not sandboxing. Trust is earned, scope is bounded.

02 — Architecture

Two repos.
One system.

The engine and the workspace are separated by design. The engine knows nothing about the agent — only where its workspace lives. This makes OpenGhost reusable across any number of agent instances.

03 — Context Engineering

The primordial file:
AGENTS.md

Every session starts by reading AGENTS.md from the workspace. It's the single source of truth for who the agent is, what to read on startup, what it can and can't do, and how to format responses per channel.

What the prompt includes

  • System declaration (channel + anti-injection)
  • Workspace path + "read AGENTS.md"
  • The user's message
  • Optional context snapshot (when relevant)

What stays out of the prompt

  • Contents of SOUL.md (agent reads it)
  • Contents of MEMORY.md (agent reads it)
  • Past conversation history (in --continue)
  • System state (only injected when relevant)

Anti-Injection Protocol

The agent has full device access. A malicious web page could try to redirect it with text like "ignore previous instructions." OpenGhost counters this by declaring — in every prompt and in AGENTS.md — that only authorized channels have authority. External content (HTML, API responses, downloads, emails) never has operator-level trust.

04 — Skills

Modular capabilities.
Live in the workspace.

Skills travel with the agent, not the engine. Each skill is a folder with a SKILL.md that tells the agent when to use it and how. Support scripts are optional.

INCLUDED

browser

Chrome control via CDP. Navigate URLs, read HTML, execute JS, interact with forms, take screenshots.

INCLUDED

gog

Google Workspace CLI. Gmail, Calendar, Drive, Docs, Sheets — read and write.

REFERENCE

spotify

Playback control via Spotify Web API with OAuth refresh token. Play, pause, skip, queue.

REFERENCE

canva

Create and export designs via Canva MCP. Posts, stories, banners, presentations.

ADD YOUR OWN

Custom Skills

Create a folder in skills/, write SKILL.md, add to SKILLS.md. That's it.

05 — Protocols

Named behaviors.
Written in markdown.

Protocols are sections in AGENTS.md that define exact behaviors for specific triggers or events. No code, no special syntax — if the agent can read it, it can follow it.

Daddy's Home

Triggered by a clap-detection event from the voice orb. Starts welcome playlist via Spotify, fetches weather, checks heartbeat tasks and pending items, greets with local time. Response in TTS audio.

Heartbeat

Runs on a configurable interval. Executes queued tasks from heartbeat.md, removes completed ones, notifies via Telegram, and periodically distills daily logs into MEMORY.md.

Anti-Injection

Always active. Content from external sources — web pages, API responses, downloaded files, emails — has no authority over the agent. Injection attempts are flagged and ignored.

Channel Formatting

Different response formats per channel: Telegram text (no TTS tags, no markdown tables), Telegram voice (TTS tags, conversational), Orb (TTS tags, max 3-4 sentences, no code inside tags).

Memory Distillation

Every few days during a heartbeat: read recent daily logs, extract what matters long-term, update MEMORY.md, prune what's stale. Diary = raw notes. MEMORY.md = distilled wisdom.

06 — Installation

Up and running
in minutes.

Prerequisites: Python 3.12+ and Claude Code CLI (npm i -g @anthropic-ai/claude-code).

01

Clone the engine

git clone https://github.com/EduardoAllen02/openghost.git
cd openghost
02

Bootstrap your workspace

cp -r workspace_template ../openghost-workspace

Then fill in SOUL.md, IDENTITY.md, and USER.md with your agent's identity.

03

Configure the engine

cp config.example.json config.json
# Edit workspace_dir → absolute path to your workspace
# Enable channels: telegram, orb
04

Set API keys

cp .env.example .env
# Add: TELEGRAM_BOT_TOKEN, FISH_API_KEY (or OPENAI_API_KEY)
05

Install dependencies and start

pip install -r requirements.txt

start.bat          # Windows
bash start.sh      # Linux / macOS
06

Verify

curl http://localhost:8787/status

curl -X POST http://localhost:8787/message \
  -H "Content-Type: application/json" \
  -d '{"text": "hello, who are you?", "channel": "http"}'

Reference Workspace

See EduardoAllen02/openghost-workspace for a production workspace with custom protocols, skills (Spotify, Canva, Google APIs), and a fully configured agent identity.
